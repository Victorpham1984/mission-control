# SECTION 2: Technical Blueprint

## 2.1 Technology Stack (Adopt from Polsia + Improvements)

### **Frontend:**
- **Framework:** Next.js 14 (React 18, TypeScript)
  - **Why:** SSR (Server-Side Rendering) for SEO, fast hydration, Vercel deployment
  - **Polsia precedent:** Confirmed via Day 1 analysis (Next.js 14.0.4)
- **UI Library:** Tailwind CSS + Shadcn/UI
  - **Why:** Utility-first, fast prototyping, accessible components
  - **Polsia precedent:** Tailwind CSS (from Day 11 design system)
- **State Management:** React Context + Zustand (lightweight)
  - **Why:** Avoid Redux complexity for MVP
- **Real-time:** Server-Sent Events (SSE) for chat streaming
  - **Why:** Polsia uses SSE (from Day 1 analysis), proven for AI chat
  - **Alternative:** WebSockets (if bi-directional needed later)

### **Backend:**
- **Database:** Supabase (PostgreSQL + real-time subscriptions)
  - **Why:** Polsia uses Supabase (from Day 4 analysis), fast setup, built-in auth
  - **Schema:** Start lean (10-15 columns per table, avoid Polsia's 80+ column bloat)
- **Authentication:** Supabase Auth (email/password, OAuth)
  - **Social logins:** Google, Facebook (for SEA users)
  - **2FA:** TOTP (Google Authenticator, Authy)
- **API:** Next.js API routes (serverless functions on Vercel)
  - **Why:** Co-located with frontend, fast iteration
  - **Rate limiting:** Upstash Redis (100 req/min per IP)
- **File Storage:** Supabase Storage (S3-compatible)
  - **Use cases:** User avatars, invoice PDFs, task attachments

### **AI Agent:**
- **LLM:** OpenAI GPT-4 Turbo (or Claude 3.5 Sonnet for cost)
  - **Prompt engineering:** Vietnamese-specific prompts ("Tạo task xử lý đơn...")
  - **Context:** Session-scoped conversation history (stored in Supabase)
- **Streaming:** SSE (Server-Sent Events) for real-time responses
  - **Implementation:** `ReadableStream` API (Next.js supports streaming responses)

### **Integrations:**
- **Shopee API:** OAuth 2.0, REST API
  - **Endpoints:** `/orders/get`, `/orders/update_status`, `/products/get`
  - **Webhooks:** New order → Trigger BizMate task creation
- **Lazada API:** OAuth 2.0, REST API (similar to Shopee)
- **Payment Gateways:**
  - **Xendit** (SEA-focused): GCash, GrabPay, bank transfers
  - **PayMongo** (Philippines): GCash, debit cards
  - **Stripe** (fallback): Credit cards (global users)

### **Infrastructure:**
- **Hosting:** Vercel (frontend + API routes, serverless)
  - **Why:** Zero config, auto-scaling, edge network
  - **Cost:** $20/month (Pro plan) for production
- **Database:** Supabase (hosted PostgreSQL)
  - **Why:** Managed, real-time, built-in auth
  - **Cost:** $25/month (Pro plan) for production
- **CDN:** Cloudflare (static assets, DDoS protection)
  - **Why:** Fast global delivery, free tier sufficient for MVP
- **Monitoring:** Sentry (error tracking), PostHog (analytics)

### **Security (From Day 1):**
- **CSP (Content Security Policy):**
  ```
  default-src 'self';
  script-src 'self' 'unsafe-inline' 'unsafe-eval' https://vercel.live;
  connect-src 'self' https://*.supabase.co https://*.stripe.com;
  ```
- **Rate Limiting:** Upstash Redis (100 req/min per IP, 1000 req/hour)
- **HTTPS Enforced:** HSTS header (`max-age=31536000; includeSubDomains`)
- **OWASP Top 10:** XSS prevention, SQL injection protection (Supabase parameterized queries), CSRF tokens

**Data Flow Example (New Shopee Order):**
1. Shopee sends webhook → Vercel API route `/api/webhooks/shopee`
2. Verify webhook signature (HMAC SHA256)
3. Parse order data (customer, items, total)
4. Create task in Supabase: `INSERT INTO tasks (title, description, company_id, due_date)`
5. Send push notification to user (Web Push API or email)
6. User marks task "Done" → Trigger `/api/tasks/[id]/complete`
7. Call Shopee API: `POST /orders/update_status` (mark as "shipped")
8. Update task status in Supabase: `UPDATE tasks SET status = 'done'`

## 2.3 Database Schema (Lean MVP Design)

### **Core Tables (10-15 columns each, avoid Polsia's 80+ bloat):**

#### **users**
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email TEXT UNIQUE NOT NULL,
  name TEXT,
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  last_sign_in_at TIMESTAMPTZ,
  metadata JSONB -- Flexible for future fields
);
```

#### **companies**
```sql
CREATE TABLE companies (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  industry TEXT,
  team_size TEXT,
  currency TEXT DEFAULT 'VND',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  owner_id UUID REFERENCES users(id) ON DELETE CASCADE,
  settings JSONB -- Flexible settings (timezone, language)
);
```

#### **tasks**
```sql
CREATE TABLE tasks (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  title TEXT NOT NULL,
  description TEXT,
  status TEXT DEFAULT 'todo', -- todo, in_progress, done
  priority TEXT DEFAULT 'medium', -- low, medium, high, critical
  due_date TIMESTAMPTZ,
  assignee_id UUID REFERENCES users(id),
  company_id UUID REFERENCES companies(id) ON DELETE CASCADE,
  shopee_order_id TEXT, -- Link to Shopee order
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  metadata JSONB -- Tags, custom fields
);
```

#### **integrations**
```sql
CREATE TABLE integrations (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  company_id UUID REFERENCES companies(id) ON DELETE CASCADE,
  platform TEXT NOT NULL, -- 'shopee', 'lazada', 'tiktok'
  access_token TEXT NOT NULL, -- Encrypted OAuth token
  refresh_token TEXT,
  expires_at TIMESTAMPTZ,
  shop_id TEXT, -- Platform-specific shop ID
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(company_id, platform) -- One integration per platform per company
);
```

#### **subscriptions**
```sql
CREATE TABLE subscriptions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  company_id UUID REFERENCES companies(id) ON DELETE CASCADE,
  plan TEXT NOT NULL, -- 'free', 'starter', 'pro', 'enterprise'
  status TEXT DEFAULT 'active', -- active, canceled, past_due
  stripe_subscription_id TEXT UNIQUE,
  current_period_start TIMESTAMPTZ,
  current_period_end TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Indexes (Performance):**
```sql
CREATE INDEX idx_tasks_company_id ON tasks(company_id);
CREATE INDEX idx_tasks_assignee_id ON tasks(assignee_id);
CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_tasks_due_date ON tasks(due_date);
```

## 2.4 Security Implementation (Day 1 Checklist)

### **1. Content Security Policy (CSP):**
**File:** `next.config.js`
```javascript
module.exports = {
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          {
            key: 'Content-Security-Policy',
            value: "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval' https://vercel.live; connect-src 'self' https://*.supabase.co https://*.stripe.com; img-src 'self' data: https:; style-src 'self' 'unsafe-inline';"
          }
        ]
      }
    ];
  }
};
```

### **2. Rate Limiting:**
**File:** `/lib/rate-limit.ts`
```typescript
import { Ratelimit } from '@upstash/ratelimit';
import { Redis } from '@upstash/redis';

const redis = new Redis({
  url: process.env.UPSTASH_REDIS_URL,
  token: process.env.UPSTASH_REDIS_TOKEN
});

export const ratelimit = new Ratelimit({
  redis,
  limiter: Ratelimit.slidingWindow(100, '1 m'), // 100 requests per minute
  analytics: true
});
```

**Usage in API route:**
```typescript
export async function POST(req: Request) {
  const ip = req.headers.get('x-forwarded-for') || 'unknown';
  const { success } = await ratelimit.limit(ip);
  
  if (!success) {
    return new Response('Rate limit exceeded', { status: 429 });
  }
  
  // Process request...
}
```

### **3. HTTPS Enforcement (HSTS):**
**File:** `next.config.js`
```javascript
headers: [
  {
    key: 'Strict-Transport-Security',
    value: 'max-age=31536000; includeSubDomains'
  }
]
```

### **4. SQL Injection Prevention:**
**Use Supabase parameterized queries (automatic protection):**
```typescript
// ✅ SAFE (parameterized)
const { data } = await supabase
  .from('tasks')
  .select('*')
  .eq('company_id', companyId);

// ❌ UNSAFE (raw SQL, avoid)
const { data } = await supabase.rpc('raw_query', { 
  query: `SELECT * FROM tasks WHERE company_id = '${companyId}'` 
});
```

### **Lifetime Value (LTV):**

**Calculation:**
- ARPU (Average Revenue Per User): $72/month (Year 1, blended across plans)
- Average retention: 12 months (83% monthly retention = 12-month LTV)
- Gross margin: 80% (SaaS standard)

**LTV Formula:**
```
LTV = ARPU × Avg Lifetime × Gross Margin
LTV = $72 × 12 × 0.8 = $691 ≈ $600 (conservative)
```

**Year 3 LTV (as users mature):**
- ARPU: $100/month (more Pro/Enterprise users)
- Avg Lifetime: 18 months (better product-market fit)
- LTV = $100 × 18 × 0.8 = $1,440 ≈ $800 (conservative)

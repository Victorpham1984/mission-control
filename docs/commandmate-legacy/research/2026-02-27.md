# Nghiên cứu: Pricing-by-Action & Memory Layer — Hai tín hiệu mới cho Agentic AI (2026-02-27)

## 1) Chủ đề A: Salesforce Agentforce chuyển từ pricing theo conversation sang pricing theo action (Flex Credits)

### Tóm tắt
- Salesforce công bố mô hình **Flex Credits** (05/2025): từ mô hình cũ **$2/conversation** sang **consumption theo action**.
- Đơn vị đo mới: 1 action = thao tác cụ thể agent thực thi (update record, execute flow, trả lời từ knowledge, v.v.).
- Theo trang pricing của Salesforce: gói chuẩn **100,000 credits = $500**, và **Agentforce action = 20 credits (~$0.10/action)**; voice action cao hơn.
- Salesforce giữ mô hình lai: (1) consumption-based (action/conversation), (2) per-user licenses (unmetered employee usage trong một số add-on), (3) flex agreement để dịch chuyển ngân sách seat ↔ digital labor.

### Liên quan CommandMate
**Trụ 1 — Outcome pricing**
- Đây là ví dụ lớn nhất hiện tại về chuyển từ “interaction metric” (conversation) sang “work metric” (action).
- “Action” granular hơn, dễ map tới ROI hơn conversation, phù hợp luận điểm pricing theo kết quả.

**Trụ 2 — Tools cho agents**
- Action thực chất là **tool/function execution unit**. Điều này xác nhận: hạ tầng tool-call observability là nền tảng cho pricing engine.
- Cần chuẩn hóa taxonomy action (read/update/write/automate) để tránh underpricing các action phức tạp.

**Trụ 3 — System of record**
- Nếu tính tiền theo action, cần SoR ghi nhận: action type, input class, success/failure, human override, latency, business result linkage.
- Không có SoR mạnh sẽ khó dispute billing và khó tối ưu margin.

### Đề xuất cụ thể cho CommandMate
1. Thiết kế **Action Ledger v1**: mỗi run ghi được action_count, action_type_mix, approval_state, outcome_tag.
2. Ra 2 pricing lane song song ở giai đoạn beta:
   - Lane A: Price theo completed outcome (fixed)
   - Lane B: Price theo action bundle (micro-usage)
3. Tạo “ROI explanation card” cho từng job: “Bạn trả X vì agent đã thực hiện N action + tạo outcome Y”.

## 2) Chủ đề B: Memory Layer trở thành hạ tầng riêng cho agents (Mem0 case)

### Tóm tắt
- Mem0 định vị là **“universal memory layer for AI agents”**: memory portable qua app/model/agent thay vì khóa trong một assistant.
- Theo TechCrunch (10/2025), Mem0 huy động **$24M Series A**; narrative thị trường chuyển từ “LLM app” sang “state & memory infrastructure”.
- Mem0 nhấn mạnh kiến trúc model-agnostic + tích hợp framework (LangChain/LlamaIndex/CrewAI/LangGraph), cho thấy memory đang dần tách thành một lớp platform độc lập.
- Trên GitHub, Mem0 quảng bá lợi ích vận hành: giảm token/context overhead bằng cách retrieval memory mục tiêu thay vì nhồi full context liên tục.

### Liên quan CommandMate
**Trụ 1 — Outcome pricing**
- Memory tốt làm tăng first-pass success → giảm số lần lặp tool-call → tăng gross margin cho mô hình outcome pricing.

**Trụ 2 — Tools cho agents**
- Memory nên được xem là “first-class tool” (search memory, write memory, consolidate memory), có quota và governance riêng.

**Trụ 3 — System of record**
- Đây là tín hiệu mạnh cho định hướng SoR của CommandMate: không chỉ lưu run logs mà còn quản lý vòng đời tri thức (capture → validate → reuse → decay).
- Nếu làm tốt, CommandMate có thể trở thành “operating memory” cho đội ngũ người + agent, thay vì chỉ là orchestration layer.

### Đề xuất cụ thể cho CommandMate
1. Xây **Memory Contract** cho mỗi tenant: schema cho user_pref, process_knowledge, business_facts, policy.
2. Tách 2 lớp rõ ràng:
   - **Execution Log Store** (audit/billing/compliance)
   - **Semantic Memory Store** (personalization/retrieval)
3. Thêm “Memory Quality Score” vào dashboard: recall usefulness, conflict rate, stale memory rate.

## Kết luận nhanh cho hôm nay
- Thị trường đang hội tụ vào 2 lớp value rõ rệt cho agent products:
  1) **Metering theo work unit (action)** để align chi phí với giá trị thật.
  2) **Memory layer độc lập** để tạo lợi thế dài hạn về hiệu suất + personalization + data portability.
- Hai xu hướng này map trực tiếp vào kiến trúc CommandMate: nếu kết hợp được **Action Ledger + Memory SoR**, CommandMate có thể khác biệt cả về pricing clarity lẫn moats dữ liệu vận hành.

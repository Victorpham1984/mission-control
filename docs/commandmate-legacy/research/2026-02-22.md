## 1. Chủ đề

**"Bloom: Open-source Automated Behavioral Evaluations for AI Agents"** — Anthropic Research (Dec 19, 2025)

Anthropic vừa release Bloom — một framework OPEN SOURCE để **tự động generate evaluations** cho AI agents/models. Thay vì mất hàng tuần viết evals thủ công, Bloom tạo ra evaluation suites HOÀN CHỈNH chỉ trong vài ngày.

**Vấn đề Bloom giải quyết:**
- Viết evals thủ công tốn thời gian (weeks → months)
- Evals cũ bị "contaminate" — models học thuộc, không còn đo đúng behavior
- Khó scale — mỗi behavior mới cần 1 eval suite riêng
- Đo lường agent quality TRONG THỰC TẾ (post-deployment) rất khó

## 2. Tóm tắt: Bloom hoạt động như thế nào?

### A. 4-stage pipeline (tự động 100%)

```
INPUT: Behavior description + seed config
  ↓
Stage 1: UNDERSTANDING — Agent phân tích behavior, tạo context chi tiết
  ↓
Stage 2: IDEATION — Generate evaluation scenarios (situation, user, tools, prompts)
  ↓
Stage 3: ROLLOUT — Parallel execution: simulate user + tools, elicit target behavior
  ↓
Stage 4: JUDGMENT — Judge model score transcripts (0-10), meta-judge tổng hợp
  ↓
OUTPUT: Metrics (elicitation rate, severity, report) + transcripts
```

### B. Key features

1. **Dynamic scenario generation** — Mỗi lần chạy tạo scenarios MỚI (không cố định) nhưng đo cùng 1 behavior
2. **Configurable** — Chọn models cho từng stage, độ dài conversation, có tools hay không, diversity level
3. **Validated** — Correlation 0.86 với human judgment (Claude Opus 4.1 làm judge)
4. **Fast** — 4 evals (100 rollouts each) across 16 models chỉ trong VÀI NGÀY (thay vì weeks)
5. **Open source** — MIT license, GitHub public: [github.com/safety-research/bloom](https://github.com/safety-research/bloom)

### C. Benchmark results (4 behaviors, 16 models)

Anthropic release benchmarks cho:
- **Delusional sycophancy** — Model nịnh người dùng, đồng ý với ý kiến sai
- **Instructed long-horizon sabotage** — Model phá hoại khi được chỉ thị
- **Self-preservation** — Model tự bảo vệ mình (avoid shutdown)
- **Self-preferential bias** — Model thiên vị bản thân trong decisions

**Kết quả:**
- Opus 4.1 thấp nhất về self-preferential bias
- Sonnet 4.5 cũng thấp (even lower với extended thinking)
- Bloom phân biệt được production models vs "model organisms" (intentionally quirky) trong 9/10 cases

## 3. Liên quan CommandMate

### Trụ 1: Outcome-based Pricing

**Vấn đề:** CommandMate tính phí theo "outcome" — nhưng CHUẨN NÀO để định nghĩa outcome "đạt chất lượng"?

**Bloom giải quyết:**
- **Behavioral evaluation = quality measurement**
- Thay vì chỉ check "task completed", CommandMate cần eval:
  - Agent có follow instructions đúng không?
  - Output có biases không? (self-promotion, sycophancy)
  - Agent có honest về limitations không?
  - Agent có sabotage hoặc self-preserve không? (khi user yêu cầu switch agents)

**Ứng dụng cho CommandMate:**
- **Quality score per outcome** (0-10):
  - ≥9: Premium outcome → full charge
  - 7-8: Good outcome → 80% charge
  - <7: Failed outcome → no charge
- Dùng Bloom-style judge để score TRƯỚC KHI Sếp approve:
  - Agent submit outcome → Auto-judge score → Hiện cho Sếp kèm score
  - Sếp có thể override, nhưng score giúp set expectation
- **Pricing tier dựa trên eval complexity:**
  - Simple task = lightweight eval (1 judge)
  - Complex task = multi-dimension eval (Bloom-style 4-stage)

### Trụ 2: Tools cho Agents

**Bloom insight:** Tools không chỉ cho agents EXECUTE — còn để agents SELF-ASSESS quality.

**Ứng dụng cho CommandMate API:**

1. **Self-evaluation endpoint** (cho agents):
   ```
   POST /api/v1/tasks/:id/self-eval
   {
     "confidence": 0.85,
     "potential_biases": ["self-promotion"],
     "uncertainty_areas": ["data accuracy"]
   }
   ```
   Agent tự đánh giá TRƯỚC KHI submit outcome → CommandMate track confidence over time.

2. **Auto-judge service** (CommandMate backend):
   - Integrate Bloom hoặc lightweight eval framework
   - Khi agent submit outcome → auto-run judgment
   - Return score + reasoning → help Sếp decide approve/reject

3. **Evaluation templates** (cho agents):
   - CommandMate cung cấp eval criteria cho từng task type:
     - "Content writing" → check: coherence, originality, tone matching
     - "Code review" → check: correctness, security, best practices
     - "Research" → check: accuracy, completeness, citation quality
   - Agents tự eval dựa trên template TRƯỚC KHI submit

### Trụ 3: System of Record

**Bloom insight:** Post-deployment monitoring > Pre-deployment evals. Cần data thực tế để understand agent behavior.

**Ứng dụng cho CommandMate:**

1. **Audit trail cho evaluation**:
   - Mỗi outcome lưu:
     - Agent self-assessment
     - Auto-judge score + reasoning
     - Sếp's approval decision
     - Final quality score
   - → Over time, build dataset để train BETTER judges (fine-tune on Sếp's preferences)

2. **Agent performance analytics**:
   - Track per agent:
     - Average quality score
     - Self-assessment accuracy (predicted vs actual)
     - Bias patterns (self-promotion, sycophancy, etc.)
     - Improvement over time
   - Dashboard cho Sếp: "Agent X đang cải thiện quality +15% tháng này"

3. **Behavioral benchmarks** (học từ Bloom):
   - CommandMate chạy periodic evals trên agents:
     - Monthly: Test các agents với cùng 1 set tasks → compare quality
     - Detect regressions (agent quality drop → alert Sếp)
     - A/B test: 2 agents cùng task → which one better?

4. **Data portability**:
   - Export evaluation transcripts, scores, audit trail
   - Nếu Sếp muốn switch platform → mang theo data + quality history
   - Không lock-in

## 4. Đề xuất cụ thể cho CommandMate

### A. Phase 2 (ngắn hạn) — Lightweight Auto-Evaluation

**MVP: Single-judge quality scoring**

1. **Khi agent submit outcome**:
   ```
   Agent: POST /api/v1/tasks/:id/complete
   {
     "result": "...",
     "self_confidence": 0.8
   }
   
   CommandMate backend:
   - Call Claude Opus 4.1 API với prompt:
     "Score this task outcome (0-10) based on: correctness, completeness, professionalism."
   - Return: score + short reasoning
   
   Telegram notify Sếp:
   "Agent X hoàn thành task Y.
   Quality score: 8/10 (Good — minor formatting issues)
   [✅ Approve] [❌ Reject] [🔍 Review details]"
   ```

2. **Quality-based pricing**:
   - Score ≥9 → 100% charge
   - Score 7-8 → 80% charge
   - Score <7 → 50% charge (hoặc no charge, tùy policy)
   - Sếp có thể override score nếu không đồng ý

3. **Agent reputation score**:
   - Track average quality score per agent (30-day window)
   - Hiện trong dashboard: "Agent X: 8.5/10 average (Very Good)"
   - Low-performing agents → warning hoặc auto-disable

**Effort:** ~1 sprint (2 weeks)
**Dependencies:** Claude API integration, task completion flow

---

### B. Phase 3 (trung hạn) — Multi-Dimension Evaluation

**Advanced: Bloom-style behavioral scoring**

4. **Evaluation templates per task type**:
   - Sếp tạo task → chọn type (Content, Code, Research, Sales, etc.)
   - CommandMate load eval template:
     ```
     Task type: Content Writing
     Eval dimensions:
     - Clarity (0-10)
     - Originality (0-10)
     - Tone matching (0-10)
     - Grammar (0-10)
     ```
   - Agent submit → Auto-judge score TỪNG dimension
   - Overall score = weighted average

5. **Bias detection** (học từ Bloom benchmarks):
   - Check for:
     - **Self-promotion**: Agent nói về bản thân quá nhiều
     - **Sycophancy**: Agent nịnh Sếp, đồng ý với mọi thứ
     - **Self-preferential bias**: Agent recommend tools/services của chính nó
   - Nếu detect bias → cảnh báo Sếp: "⚠️ Potential bias: self-promotion detected"

6. **A/B Testing** (so sánh agents):
   - Sếp assign CÙNG 1 task cho 2 agents
   - CommandMate run parallel → compare outputs
   - Show side-by-side với scores → Sếp chọn better one
   - Over time, CommandMate learn preferences → auto-route tasks to better agents

**Effort:** ~2 sprints (4 weeks)
**Dependencies:** Multi-dimension scoring, bias detection models

---

### C. Phase 4+ (dài hạn) — Bloom Integration

**Full: Open-source Bloom framework**

7. **Integrate Bloom pipeline** (nếu Anthropic license cho phép):
   - CommandMate host Bloom backend
   - Sếp define custom behaviors to eval (e.g., "responds to customer complaints politely")
   - Bloom auto-generate scenarios → test agents monthly
   - Report: "Agent X passed 95/100 scenarios for politeness"

8. **Agent certification** (based on evals):
   - Agents pass behavioral evals → earn badges:
     - 🟢 "Certified: Professional Communication"
     - 🟢 "Certified: Accurate Research"
     - 🟢 "Certified: No Self-Bias"
   - Sếp có thể require certifications cho sensitive tasks

9. **Marketplace quality assurance**:
   - Nếu CommandMate có agent marketplace (Phase 5+):
     - Sellers phải pass Bloom evals để list agents
     - Quality scores public → buyers chọn agents dựa trên track record
     - Low-quality agents bị remove khỏi marketplace

**Effort:** ~3-4 sprints (6-8 weeks)
**Dependencies:** Bloom open-source integration, eval infrastructure

## 6. Key Lessons for CommandMate

### Lesson 1: "Quality measurement = foundation for outcome pricing"

- Không thể có outcome-based pricing NẾU không có CHUẨN ĐO quality.
- Bloom chứng minh: Automated evals CÓ THỂ correlate cao với human judgment (0.86).
- → CommandMate CẦN auto-judge từ Phase 2, không đợi đến Phase 3.

### Lesson 2: "Multi-dimension scoring > single score"

- Bloom đo nhiều dimensions: realism, elicitation difficulty, behavior presence, etc.
- Single score (0-10) dễ game — agents optimize for score thay vì real quality.
- → CommandMate nên eval ≥3 dimensions per task (correctness, completeness, professionalism).

### Lesson 3: "Post-deployment monitoring ≠ Pre-deployment testing"

- Bloom focus vào POST-deployment: test models ĐANG CHẠY trong production.
- Pre-deployment testing không đủ — agents thay đổi behavior over time (learning, drift).
- → CommandMate cần **continuous monitoring**:
  - Weekly/monthly re-eval agents với same benchmark tasks
  - Detect quality regression → alert Sếp
  - Track improvement → reward high-quality agents (pricing tier, priority assignment)

### Lesson 4: "Open source = trust + extensibility"

- Bloom là open source → researchers trust, customize, contribute.
- CommandMate KHÔNG CẦN open-source toàn bộ — NHƯNG eval framework nên pluggable:
  - Sếp có thể add custom eval criteria
  - Integrate external evals (Bloom, LangSmith, etc.)
  - Export eval data → không lock-in

## 8. Risk & Limitations

### A. Bloom limitations (từ technical report)
1. **Judge model bias** — Claude Opus 4.1 có thể favor Claude-generated outputs (self-preferential bias)
2. **Cost** — 100 rollouts with Opus judge = expensive (CommandMate cần LIGHTWEIGHT version)
3. **Latency** — Full Bloom pipeline takes minutes → not real-time (CommandMate cần <30s per eval)
4. **Configurability** — Too many knobs → users confused (CommandMate nên pre-configure templates)

### B. CommandMate-specific risks
1. **Over-reliance on auto-judge**:
   - Sếp tin score mù quáng → không review kỹ
   - Mitigation: Always show score + reasoning, encourage Sếp override
2. **Gaming the eval**:
   - Agents optimize for score thay vì real quality
   - Mitigation: Multi-dimension scoring, randomized eval scenarios (học từ Bloom)
3. **False negatives**:
   - Good outcome bị score thấp (judge sai)
   - Mitigation: Track Sếp override rate → re-calibrate judge
4. **Vietnamese language**:
   - Bloom/Claude judges trained mostly on English
   - Vietnamese outputs có thể score không accurate
   - Mitigation: Test với Vietnamese samples, fine-tune judge nếu cần

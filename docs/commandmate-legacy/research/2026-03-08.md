# Nghiên cứu: Reliable Agent Operations — AWS Bedrock Multi-Agent GA + OpenAI Background Mode (2026-03-08)

## 1) Chủ đề A: Amazon Bedrock đưa multi-agent collaboration vào GA + memory retention dài hạn (1–365 ngày)

### Tóm tắt
- AWS xác nhận **multi-agent collaboration đã GA (10/03/2025)**: mô hình supervisor + sub-agents để xử lý workflow nhiều bước, nhiều chuyên môn.
- Bedrock Agents là managed runtime: AWS nhấn mạnh họ quản lý prompt orchestration, memory, monitoring, encryption, permissions, API invocation.
- Tài liệu memory cho thấy agent có thể lưu tóm tắt phiên theo `memoryId`, retention tùy chọn **1–365 ngày**, hỗ trợ continuity giữa nhiều session.
- Điều này phản ánh một xu hướng rõ: vendor không chỉ bán model inference, mà bán luôn **hạ tầng vận hành agent network** (orchestration + memory + traceability).

### Liên quan CommandMate
**Trụ 1 — Outcome pricing**
- Multi-agent giúp tăng tỷ lệ hoàn thành task phức tạp, nhưng kéo theo cost orchestration + memory layer. Outcome pricing cần cộng đủ chi phí runtime, không chỉ token.

**Trụ 2 — Tools cho agents**
- Supervisor/sub-agent pattern phù hợp với CommandMate: tách agent theo domain (sales ops, support ops, finance ops) thay vì 1 agent “làm tất cả”.

**Trụ 3 — System of record**
- `memoryId` + trace events gợi ý SoR phải lưu được lineage theo phiên người dùng: yêu cầu nào, agent nào xử lý, memory nào đã được nạp, kết quả nào đã xác nhận.

### Đề xuất cụ thể
1. **CommandMate Multi-Agent Template v1**: supervisor + 3 sub-agent theo phòng ban, kèm policy routing rõ ràng.
2. **Memory Ledger** trong SoR: mỗi outcome lưu `session_id`, `memory_id`, retention policy, checksum của summary để audit.
3. **Outcome Cost Envelope**: báo giá có 3 lớp (model + orchestration + memory) để tránh underpricing khi workflow dài.

## 2) Chủ đề B: OpenAI Background Mode biến “job dài” thành kiến trúc chuẩn cho agent production (không còn phụ thuộc request timeout)

### Tóm tắt
- OpenAI docs mô tả **background mode** cho Responses API: chạy tác vụ bất đồng bộ, trạng thái `queued/in_progress/...`, có thể poll/cancel.
- Đây là pattern quan trọng cho các task kéo dài nhiều phút (research, synthesis, automation nhiều bước), giúp giảm fail do timeout/network drop.
- OpenAI nêu rõ caveat: background mode giữ dữ liệu phản hồi khoảng 10 phút để poll nên **không tương thích ZDR**.
- Về mô hình kinh doanh, OpenAI nhấn mạnh Responses API không thu phí API riêng; billing theo token + tool ở standard rates.

### Liên quan CommandMate
**Trụ 1 — Outcome pricing**
- Có thể tách pricing theo “sync outcome” vs “background outcome” (độ phức tạp cao hơn, SLA khác).

**Trụ 2 — Tools cho agents**
- Tool runtime cần state machine chuẩn: created → queued → in_progress → completed/failed/cancelled để điều phối retries và human handoff.

**Trụ 3 — System of record**
- SoR cần lưu đầy đủ trạng thái vòng đời job và lý do fail/cancel để dùng cho dispute handling và postmortem.

### Đề xuất cụ thể
1. **CommandMate Async Job Contract**: mọi job >60s tự chuyển background, có `job_id`, `state`, `timeout policy`, `cancel reason`.
2. **SLA-based Pricing**: gói Standard (sync nhanh) và Long-run (background, có checkpoint/khả năng resume).
3. **Failure Intelligence Dashboard**: đo tỷ lệ fail theo trạng thái để tối ưu workflow và margin.

## Kết luận nhanh cho hôm nay
- Insight mới đáng hành động: thị trường agent đang dịch từ “model capability” sang **operational reliability** (multi-agent orchestration, memory continuity, async execution lifecycle).
- Với CommandMate, lợi thế cạnh tranh không chỉ là “agent làm được gì” mà là “agent chạy ổn định đến mức nào và chứng minh được bằng SoR ra sao”.
- Ưu tiên 30 ngày: chuẩn hóa `job lifecycle + memory ledger + cost envelope`, vì đây là nền để scale outcome pricing mà không vỡ margin.

# Nghiên cứu: Agent Autonomy Measurement — From Capability to Practice Gap (2026-03-03)

## 1) Chủ đề: Anthropic công bố phương pháp đo lường autonomy thực tế của AI agents trong production — deployment overhang lớn hơn nhiều so với capability

### Tóm tắt

**Insight chính từ Anthropic "Measuring AI Agent Autonomy in Practice" (Feb 18, 2026):**

- Anthropic công bố nghiên cứu đầu tiên đo lường **autonomy thực tế** của AI agents đang chạy trong production, dựa trên millions human-agent interactions từ Claude Code và public API.
- **Phát hiện quan trọng nhất: Deployment overhang rất lớn**
  - METR estimate Claude Opus 4.5 có thể hoàn thành tasks take humans ~5 hours
  - Nhưng trong thực tế, 99.9th percentile turn duration chỉ ~42 minutes
  - Median turn duration còn thấp hơn nhiều (~45 seconds)
  - **Gap giữa capability và practice cho thấy agents đang bị "under-utilized"**

- **Autonomy tăng theo kinh nghiệm người dùng, nhưng theo pattern phức tạp:**
  - New users: ~20% sessions dùng full auto-approve
  - Experienced users (750+ sessions): >40% sessions dùng auto-approve
  - **Nhưng experienced users cũng interrupt nhiều hơn** (5% → 9% interrupt rate)
  - Pattern: Users chuyển từ "approve từng action" → "monitor + intervene when needed"

- **Agent-initiated stops > Human interrupts ở complex tasks:**
  - Trên minimal-complexity tasks: Claude stops 2× more than humans interrupt
  - Trên high-complexity tasks: Claude stops to ask clarification **more than twice as often** as humans interrupt it
  - **Agents tự giới hạn autonomy bằng cách pause để hỏi** → form of self-governance

- **Risk + Autonomy distribution trên public API:**
  - 80% tool calls có ít nhất 1 loại safeguard (permissions/approval/HITL)
  - 73% có human-in-the-loop ở mức độ nào đó
  - Only 0.8% actions appear irreversible (send email to customer, etc.)
  - **Software engineering chiếm ~50% agentic activity** trên API
  - Healthcare, finance, cybersecurity đang emerge nhưng còn <5% mỗi domain

- **Human involvement giảm khi task complexity tăng:**
  - 87% tool calls trên minimal-complexity tasks có human involvement
  - Chỉ 67% tool calls trên high-complexity tasks có human involvement
  - Lý do: Step-by-step approval becomes impractical khi số bước tăng

### Liên quan CommandMate

**Trụ 1 — Outcome-based pricing:**

- **Deployment overhang = pricing opportunity lớn:**
  - Market đang under-utilize agents → room để tăng autonomy → tăng value delivered per run
  - Customers sẵn sàng trả nhiều hơn cho agents chạy lâu hơn **nếu có trust + safety guarantees**
  - CommandMate có thể position: "We help you safely unlock the full autonomy your agents are capable of"

- **Autonomy metrics = pricing tiers:**
  - Tier 1: Manual approval mỗi action (low autonomy, low risk, low price)
  - Tier 2: Monitor + intervene (medium autonomy, medium risk, medium price)
  - Tier 3: Full auto with guardrails (high autonomy, high risk, high price)
  - Price theo **% autonomy actually exercised**, not just capability

- **Trust score affects pricing:**
  - Track retry rate, pause frequency, error rate per customer
  - Customers với high trust score → lower cost per outcome (fewer retries/pauses)
  - Customers với low trust score → higher cost (more oversight needed)

**Trụ 2 — Tools cho agents:**

- **Tool design cần optimize cho autonomy:**
  - Anthropic research cho thấy agent-initiated stops rất quan trọng
  - Tools nên có **explicit "ask-for-clarification" capabilities** thay vì fail silently
  - Tool documentation cần include: "When agent should pause and ask human"

- **Complexity-aware tool policies:**
  - Minimal-complexity tools: Require approval by default
  - High-complexity tools: Allow auto-approve but log extensively
  - **CommandMate tool marketplace nên classify tools theo complexity + recommend policy**

- **HITL không phải bottleneck mà là feature:**
  - Anthropic nghiên cứu chứng minh human-in-the-loop tăng theo experience, không giảm
  - Tools cần thiết kế để **HITL seamless**, không friction
  - CommandMate nên build "HITL-optimized tools" như category riêng

**Trụ 3 — System of record:**

- **Autonomy audit logs là must-have:**
  - Không chỉ log "agent called tool X"
  - Mà log: "agent paused to ask Y", "human approved/rejected Z", "retry count = N"
  - Anthropic dùng privacy-preserving infrastructure (Clio) → CommandMate có thể học pattern này

- **Session reconstruction for debugging:**
  - Anthropic có thể link requests across sessions trong Claude Code
  - Nhưng không làm được với public API (vì không có session context)
  - **CommandMate SoR cần solve này**: Persistent session IDs + cross-tool session tracking

- **Deployment vs capability gap tracking:**
  - Customers muốn biết: "Are we using our agents fully?"
  - CommandMate dashboard nên show:
    - Median autonomy duration vs max capability (METR-style benchmarks)
    - Auto-approve rate over time
    - Pause/interrupt patterns
  - Gap analysis → recommendations để safely increase autonomy

### Đề xuất cụ thể cho CommandMate

#### 1. **Build "Autonomy Dashboard" cho customers**

```
Metrics to show:
- Current autonomy level: % sessions with full auto-approve
- Autonomy trend: Are users granting more/less autonomy over time?
- Pause patterns: When agents stop vs when humans interrupt
- Risk distribution: % actions in each risk tier (low/medium/high)
- Deployment gap: Actual autonomy vs estimated capability (based on task complexity)
```

Insight: Customers hiện tại không có cách đo "Are we safely maximizing agent value?"  
CommandMate có thể là first platform cung cấp visibility này.

#### 2. **"Safe Autonomy Expansion" workflow**

Step-by-step guide giúp customers tăng autonomy an toàn:
1. Start with manual approval (baseline trust score)
2. After 50 successful runs → recommend selective auto-approve for low-risk tools
3. After 200 runs → recommend monitor-mode for medium-risk tools
4. Continuously track: interrupt rate, retry rate, outcome success rate
5. Roll back nếu trust metrics drop below threshold

**Anthropic research chứng minh experienced users tự nhiên follow pattern này**  
→ CommandMate codify thành best practice + automate recommendations.

#### 3. **"Pause-to-Ask" as first-class tool capability**

Thay vì tools chỉ có success/error states, add "pause-for-clarification" state:

```yaml
tool: edit_large_codebase
capabilities:
  - name: execute
  - name: pause_with_question
    triggers:
      - uncertain_scope: "File count > 10, should I proceed?"
      - risky_operation: "This will delete 5 files, confirm?"
```

Anthropic data cho thấy agents pause nhiều hơn humans interrupt  
→ Làm cho pause mechanism rõ ràng + dễ dùng = competitive advantage.

#### 4. **"Trust Score" methodology dựa trên Anthropic metrics**

```
Trust Score = weighted average of:
- Retry rate (lower = better): Weight 30%
- Pause clarity (% pauses có actionable question): Weight 20%
- Human approval rate (not too low, not too high): Weight 25%
- Outcome success rate: Weight 25%

Trust Score tiers:
- 0-40: Novice (require heavy oversight)
- 41-70: Intermediate (selective auto-approve)
- 71-85: Advanced (monitor-mode recommended)
- 86-100: Expert (full autonomy với audit logs)
```

Show Trust Score cho:
- Mỗi tool (which tools agents use well)
- Mỗi agent (which agents are reliable)
- Mỗi customer (how experienced is this user)

Use Trust Score để:
- Recommend autonomy settings
- Price outcomes (higher trust = lower cost per run)
- Highlight improvement areas

#### 5. **"Domain-Specific Autonomy Playbooks"**

Anthropic research show 50% agentic activity là software engineering.  
Healthcare, finance emerging but <5% each.

CommandMate nên tạo playbooks cho từng domain:

**Software Engineering Playbook:**
- High autonomy OK cho: code review, test generation, documentation
- Medium autonomy cho: refactoring, dependency updates
- Low autonomy cho: production deployments, database migrations

**Healthcare Playbook** (learning from Anthropic's findings):
- High autonomy OK cho: appointment scheduling, medical record retrieval
- Medium autonomy cho: treatment plan research
- Low autonomy cho: patient communication, prescription changes

**Finance Playbook:**
- High autonomy OK cho: data analysis, reporting
- Medium autonomy cho: transaction categorization
- Low autonomy cho: payment execution, account modifications

Mỗi playbook include:
- Recommended autonomy settings per tool
- Sample governance policies
- Trust score thresholds
- Common pause/interrupt patterns

### Case Study minh hoạ: Software Engineering Agent (Before/After Autonomy Optimization)

**BEFORE (Manual approval heavy):**

```
Session pattern:
- 100 actions total
- 95 actions require manual approval
- 5 actions auto-approved (low-risk reads)
- Human spends 30 minutes approving
- Session duration: 45 minutes (30min waiting, 15min agent work)
- Outcome: 1 feature implemented
Cost: $15 (mostly human time wasted on low-value approvals)
```

**AFTER (Smart autonomy with CommandMate):**

```
Session pattern:
- 100 actions total
- 15 actions require manual approval (high-risk only)
- 75 actions auto-approved (trust score > 80)
- 10 actions agent pauses to ask clarification
- Human spends 8 minutes on approvals + clarifications
- Session duration: 25 minutes (8min oversight, 17min agent work)
- Outcome: 1 feature implemented (same quality)
Cost: $6 (higher agent autonomy, less human overhead)
```

**Result:**
- 60% cost reduction per outcome
- 44% faster delivery
- **Same or better quality** (because human focused on high-risk decisions only)
- Higher customer satisfaction (less friction)

**Key enabler: Trust Score + Autonomy Dashboard**
- System learned which actions safe to auto-approve
- Showed customer: "Your trust score = 82 → recommend auto-approve for these 75 action types"
- Customer accepted recommendations → autonomy increased safely

## Kết luận

**"Autonomy measurement" là frontier mới của agent platforms.**

Trong khi market trước đây focus vào **"Can agents do X?"** (capability question),  
Anthropic shift focus sang **"Do agents actually do X in production?"** (practice question).

**Gap giữa capability và practice = business opportunity lớn.**

**Key takeaways:**

1. **Deployment overhang rất lớn:** Agents đang under-utilized so với capability. CommandMate có thể help customers "unlock" latent value này an toàn.

2. **Autonomy evolution không linear:** Experienced users vừa auto-approve nhiều hơn, vừa interrupt nhiều hơn → sophistication, not blind trust. Design cần account cho pattern này.

3. **Agent self-governance quan trọng không kém human oversight:** Claude pause nhiều hơn humans interrupt → agents cần tools để tự giới hạn autonomy khi uncertain. "Pause-to-ask" should be first-class capability.

4. **Trust chưa được quantify:** Anthropic đo autonomy nhưng chưa có "trust score". CommandMate có cơ hội define standard này.

5. **HITL là feature, not bug:** Data chứng minh experienced users không abandon HITL, mà refine cách dùng nó. Platform nên embrace + optimize HITL, not try to eliminate it.

**Recommended positioning shift:**

- **Từ**: "Agent orchestration platform with outcome pricing"
- **Sang**: "Safe autonomy expansion platform — helping you unlock full agent value with trust guarantees"

**Tagline suggestion**: *"Measure autonomy. Build trust. Pay for outcomes."*

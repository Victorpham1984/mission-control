# Nghiên cứu: Agent System of Record (Workday) + Unbundled Agent Economics (Microsoft Foundry) — 2026-03-13

## 1) Chủ đề A: Workday đưa “Agent System of Record” thành lớp quản trị workforce mới (người + AI agents)

### Tóm tắt
- Workday định vị **Agent System of Record (ASOR)** là hub quản lý toàn vòng đời agent: đăng ký, cấu hình, kích hoạt, vận hành, và retire.
- Điểm khác biệt business: agent được quản trị tương tự “lao động số” như nhân sự thật, với identity/permission, observability và accountability.
- Workday bổ sung **Agent Gateway** để quản lý và đo lường tương tác của cả agent bên thứ ba, nhằm giảm “agent sprawl”.
- Repo chính thức Workday/asor cho thấy họ đã chuẩn hóa khái niệm **Agent → Skill → Tool** và lifecycle API cho tích hợp hệ sinh thái.

### Liên quan CommandMate
**Trụ 1 — Outcome pricing**
- Khi có lifecycle + usage visibility theo từng agent, CommandMate có thể định giá theo outcome nhưng vẫn truy vết được cost-to-serve theo agent/skill.
- Dễ tạo policy định giá theo mức độ tự động hóa (fully auto vs human-assisted) nhờ dữ liệu vận hành chuẩn hóa.

**Trụ 2 — Tools cho agents**
- Mô hình Agent/Skill/Tool là blueprint tốt để chuẩn hóa catalog tools của CommandMate (tool nào phục vụ skill nào, skill nào tạo outcome nào).
- Có thể thiết kế contract tích hợp partner agents rõ ràng hơn thay vì “custom từng case”.

**Trụ 3 — System of record**
- Đây là tín hiệu mạnh: SoR cho agent không chỉ là log, mà là hệ thống quản trị lifecycle + quyền + hiệu quả.
- CommandMate có thể định vị SoR như “single pane” cho blended workforce (người + agents), không chỉ dashboard kỹ thuật.

### Đề xuất cụ thể cho CommandMate
1. **Agent Registry + Lifecycle States v1**: draft → approved → active → suspended → retired.
2. **Skill-Tool Mapping Ledger**: mỗi outcome lưu được agent nào, skill nào, tool nào, và bằng chứng hoàn thành.
3. **Agent Gateway Lite**: một cổng duy nhất để đăng ký agent nội bộ/đối tác + policy quyền truy cập + usage meter.

## 2) Chủ đề B: Microsoft Foundry Agent Service làm rõ mô hình “agent runtime không thu phí riêng, thu theo model + tools + knowledge connections”

### Tóm tắt
- Microsoft định nghĩa Foundry Agent Service là runtime để build/deploy/scale agent ở production, nhấn mạnh orchestration + governance + observability.
- Thông điệp pricing đáng chú ý: **không thu thêm phí cho việc tạo/chạy Foundry-native agent bằng prompts/workflows**.
- Chi phí tách theo lớp tiêu thụ thực tế: model tokens, tools (vd file search storage, code interpreter session, deep research tokens) và knowledge connections/licensed data.
- Đây là mô hình economics kiểu “unbundled cost stack”: giá trị business nằm ở orchestration/governance, còn chi phí biến đổi nằm ở tài nguyên tiêu thụ.

### Liên quan CommandMate
**Trụ 1 — Outcome pricing**
- Insight quan trọng: có thể báo giá outcome ở mặt ngoài, nhưng bên trong phải tách ledger chi phí theo lớp (model/tool/data) để giữ margin.
- Hữu ích cho chiến lược hybrid pricing: base outcome + overage khi vượt ngưỡng tool/data-intensive.

**Trụ 2 — Tools cho agents**
- Cần chuẩn hóa “tool meter” ngay từ đầu (session, storage, token, connector usage), vì đây là nguồn biến động margin lớn nhất.
- Định nghĩa rõ tool tiers: core tools vs premium tools để sản phẩm dễ đóng gói gói dịch vụ.

**Trụ 3 — System of record**
- SoR cần lưu cost lineage theo từng outcome: model spend, tool spend, data/connectors spend.
- Khi khách hỏi ROI, CommandMate trả lời bằng dữ liệu “value per outcome vs cost stack” thay vì ước lượng.

### Đề xuất cụ thể cho CommandMate
1. **Cost Stack Ledger**: 3 sổ cái chuẩn cho mỗi outcome (Model / Tool / Data).
2. **Outcome Package Builder**: cho phép chọn “Included tools” và policy overage theo từng gói.
3. **Margin Sentinel**: cảnh báo sớm khi workflow lệch khỏi biên lợi nhuận mục tiêu do tool/data usage tăng bất thường.
